# 테스트 안내

이 디렉터리는 단위 테스트, 통합 테스트, 회귀 테스트를 두는 위치입니다.

권장 원칙:

- 소스 구조와 최대한 대응되도록 테스트를 배치합니다.
- 기능을 추가할 때 테스트도 같은 작업 안에서 추가하거나 갱신합니다.
- 실패 재현용 테스트는 문제 설명이 드러나는 이름으로 작성합니다.

시작 예시:

- `test_main.py`
- `main.test.ts`
- `app_integration_test.rs`

주의:

- 테스트 전략이 바뀌면 `docs/architecture.md`와 `README.md`를 함께 확인합니다.
- 검증을 하지 못한 경우에는 `CONTEXT.md` 또는 작업 결과에 이유를 남깁니다.
