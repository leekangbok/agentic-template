# Codex CLI 프로젝트 템플릿

이 저장소는 Codex CLI 기반으로 새 프로젝트를 시작할 때 바로 복제해서 사용할 수 있는 실전용 템플릿입니다.
목적은 프로젝트 규칙, 현재 문맥, 작업 목록, 아키텍처를 분리해 새로운 Codex 세션이 상태를 빠르게 복구하고 안전하게 작업하도록 만드는 것입니다.

## 이 템플릿이 해결하는 문제

- 프로젝트 시작 구조가 매번 달라지는 문제
- AI 세션이 충분한 문맥 없이 시작되는 문제
- 코드 변경이 문서에 반영되지 않는 문제
- 세션이 바뀔 때 설계와 진행 이력이 끊기는 문제

## 포함 구조

- `README.md`
  프로젝트 개요, 작업 흐름, 사용 방법
- `AI_RULES.md`
  Codex 및 AI 에이전트가 따라야 하는 필수 운영 규칙
- `CONTEXT.md`
  현재 프로젝트 상태, 리스크, 부족한 정보, 다음 작업
- `TASKS.md`
  우선순위가 있는 작업 목록과 완료 기준
- `.codex-start.txt`
  Codex 세션 시작용 프롬프트
- `docs/architecture.md`
  아키텍처 메모, 모듈 경계, 의사결정 기록
- `scripts/auto_codex_context.py`
  `CONTEXT.md`의 자동 생성 상태 블록을 갱신하는 스크립트
- `scripts/run.sh`, `scripts/run.bat`
  작업 시작 전 컨텍스트를 갱신하는 헬퍼 스크립트
- `src/`
  애플리케이션 소스 코드
- `tests/`
  테스트 파일

## 빠른 시작

1. 이 템플릿을 새 프로젝트 디렉터리로 복제합니다.
2. 먼저 아래 파일들을 프로젝트에 맞게 수정합니다.
   - `README.md`: 프로젝트명, 제품 목표, 명령어, 기술 스택
   - `CONTEXT.md`: 현재 상태, 리스크, 다음 작업
   - `TASKS.md`: 실제 우선순위
   - `docs/architecture.md`: 실제 모듈 경계와 설계 결정
3. 새 세션을 시작하기 전에 아래 명령으로 컨텍스트를 갱신합니다.

```sh
python scripts/auto_codex_context.py
```

4. `.codex-start.txt`를 Codex 세션 시작 프롬프트로 사용합니다.

## 권장 세션 흐름

1. `README.md`, `AI_RULES.md`, `CONTEXT.md`, `TASKS.md`를 읽습니다.
2. 현재 요청과 직접 관련된 파일만 우선 확인합니다.
3. 요청된 변경을 구현합니다.
4. 같은 작업 안에서 관련 문서도 함께 수정합니다.
5. 가능한 검증을 수행합니다.
6. `python scripts/auto_codex_context.py`를 실행합니다.
7. 다음 세션이 바로 이어받을 수 있도록 `TASKS.md`, `CONTEXT.md`를 정리합니다.

## 권장 운영 규칙

- 변경은 작게 하고 빠르게 검증합니다.
- 구조가 바뀌면 `docs/architecture.md`를 업데이트합니다.
- 우선순위가 바뀌면 `TASKS.md`를 바로 수정합니다.
- 세션 종료 전 `CONTEXT.md`가 인수인계 가능한 상태인지 확인합니다.

## 헬퍼 스크립트

Windows:

```bat
scripts\run.bat
```

macOS / Linux / WSL:

```sh
sh scripts/run.sh
```
