# 보안 메모

이 문서는 보안 관련 결정과 최소 기준을 기록합니다.

## 기본 기준

- **비밀 정보 보호**: API 키, 비밀번호 등은 절대 저장소에 커밋하지 않으며 `.env` 파일을 통해 관리한다.
- **외부 입력 불신**: 모든 외부 입력은 검증 전엔 신뢰하지 않는다.
- **최소 권한 원칙**: 애플리케이션에 필요한 최소한의 권한만 부여한다.

## 기술 스택별 보안 가이드

### Backend (Go/GORM/MySQL)
- **SQL Injection 방지**: GORM의 준비된 문장(Prepared Statements)을 사용하여 파라미터화된 쿼리를 생성한다. 날것의 SQL(Raw SQL) 사용 시 지극히 주의한다.
- **환경 변수 노출 방지**: `.env.example`을 최신화하여 필요한 변수 목록만 공개한다.

### Frontend (React)
- **XSS 방지**: 데이터 렌더링 시 React의 기본 이스케이프 기능을 신뢰하되, `dangerouslySetInnerHTML`은 절대 사용하지 않는다.
- **CSRF 방지**: 필요 시 백엔드와 연동하여 CSRF 토큰 처리를 검토한다.

## 추후 채울 내용

- JWT 인증 전략 및 토큰 만료 정책
- 사용자 권한 모델 (RBAC/ABAC)
- 민감 데이터 암호화 기준
- 보안 감사 로그 기록 기준
