# 소스 코드 안내

이 디렉터리는 실제 애플리케이션 코드를 두는 위치입니다.

권장 원칙:

- 진입점 파일과 핵심 도메인 로직을 분리합니다.
- 공통 유틸리티는 목적이 분명할 때만 추가합니다.
- 구조가 커지면 계층 또는 기능 단위 폴더로 분리합니다.

시작 예시:

- `main.py`, `index.ts`, `app.rs` 같은 첫 엔트리 파일 추가
- `domain/`, `services/`, `adapters/` 같은 구조로 확장

주의:

- 구조를 바꾸면 `docs/architecture.md`를 함께 업데이트합니다.
- 새 모듈을 추가하면 관련 테스트도 `tests/`에 추가합니다.
